> **Automatically analyze GitHub Actions build failures using a self-hosted AI model — no external API calls, no data leaving your cluster.**

When `release_build` or `snapshot_build` workflows fail in your Fabric2 GitHub repos, this system **fetches the runner logs**, **feeds them to Gemma 3 27B** running on-premises in GDC/GKE, and **posts a structured Root Cause Analysis** directly on the PR — in under 2 minutes.

---

## How It Works

```
Build fails (release_build / snapshot_build)
         │
         │  workflow_run: completed + conclusion: failure
         ▼
  rca_trigger.yml fires on self-hosted runner
         │
         ├─① Fetch failed job logs — GitHub REST API
         ├─② Parse & extract error context — pattern matching + 80K token trim
         ├─③ POST to Ollama /api/chat — Gemma 3 27B inference (on-cluster)
         ├─④ Post RCA as PR comment — root cause + recommendation
         └─⑤ Upload JSON artifact — structured report, 30-day retention
                  │
                  └─ (optional) Slack notification
```

**Sample PR comment output:**

```
🧪 Build Failure RCA — release_build

Confidence: 🟢 High  |  Category: test_failure  |  Branch: main  |  Commit: a3f7d2c1

Root Cause
AuthTokenServiceTest.testValidTokenGeneration failed because the JWT secret
environment variable is not set in the CI runner configuration.

Build Time Analysis
  Total duration:   6.2 min  ✅ (threshold: 20 min)
  Slowest step:     Run tests — 5.1 min

Failed Components
  • com.fabric2.auth.AuthTokenServiceTest#testValidTokenGeneration
  • com.fabric2.auth.AuthTokenServiceTest#testTokenRefresh

Recommendation
Add JWT_SECRET to your repository Actions secrets and reference it in
the workflow env block. See application-test.yml line 42.

🤖 Generated by Build RCA Agent (Gemma 3 27B) | View full logs
```

## Quick Start

### 1 — Install dependencies

```bash
pip install -r requirements.txt
```

### 2 — Run unit tests (no external services needed)

```bash
python -m pytest tests/ -v
```

### 3 — Test log parsing locally

```bash
python scripts/parse_logs.py --input tests/sample_logs/build_failure.log
```

### 4 — Test Ollama connection (requires running Ollama)

```bash
export OLLAMA_HOST=http://localhost:11434
python scripts/analyze_with_ollama.py --test
```

### 5 — Run the full RCA pipeline

```bash
export GITHUB_TOKEN=ghp_your_token
export OLLAMA_HOST=http://localhost:11434

python scripts/run_rca.py \
  --repo your-org/your-repo \
  --run-id 12345678 \
  --workflow-name "release_build" \
  --branch "main" \
  --sha "abc123"
```

Output is written to `rca_output/rca_report.md` and `rca_output/rca_report.json`.

## Deploy to Kubernetes (GDC/GKE)

```bash
# 1. Create namespace + storage
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/pvc.yaml

# 2. Deploy Ollama with GPU
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml

# 3. Pull Gemma 3 27B model (~16 GB, one-time)
kubectl apply -f k8s/init-model-job.yaml
kubectl logs job/ollama-model-pull -n ollama-rca -f

# 4. Verify
kubectl port-forward svc/ollama-rca -n ollama-rca 11434:11434
curl http://localhost:11434/api/tags
```

> **Node requirement**: GPU node with NVIDIA T4 or L4 (24 GB+ VRAM). Update the `nodeSelector` in `k8s/deployment.yaml` to match your GPU type.

## Add to a Fabric2 Repo

1. Copy the workflow to each target repo:

```bash
cp .github/workflows/rca_trigger.yml your-repo/.github/workflows/
```

2. Update three fields in `rca_trigger.yml`:

```yaml
# Match your actual build workflow names (case-sensitive)
workflows:
  - "release_build"
  - "snapshot_build"

# Match your enterprise runner labels
runs-on: [self-hosted, linux, gdc]

# URL your runner can reach Ollama on
OLLAMA_HOST: "http://ollama-rca.ollama-rca.svc.cluster.local:11434"
```

3. Add secrets to the repo (Settings → Secrets → Actions):

| Secret | Required | Description |
|--------|----------|-------------|
| `RCA_GITHUB_TOKEN` | Yes | PAT with `actions:read` + `pull-requests:write` |
| `SLACK_WEBHOOK_URL` | No | Slack incoming webhook URL |

4. Commit to the **default branch** (`main`/`master`) — `workflow_run` triggers only activate from the default branch.

## RCA Output Schema

The pipeline always writes a structured JSON artifact:

```json
{
  "rca": {
    "root_cause": "AuthTokenServiceTest.testValidTokenGeneration failed because JWT_SECRET is not set in CI",
    "category": "test_failure",
    "confidence": "high",
    "failed_components": [
      "com.fabric2.auth.AuthTokenServiceTest#testValidTokenGeneration",
      "com.fabric2.auth.AuthTokenServiceTest#testTokenRefresh"
    ],
    "build_time_analysis": {
      "total_duration_minutes": 6.2,
      "exceeded_threshold": false,
      "slowest_step": "Run tests",
      "slowest_step_duration_minutes": 5.1
    },
    "error_messages": [
      "java.lang.AssertionError: expected:<200> but was:<500>"
    ],
    "recommendation": "Add JWT_SECRET to Actions secrets. Reference via env: JWT_SECRET: ${{ secrets.JWT_SECRET }}",
    "additional_notes": "Two related tests failed suggesting a shared setup issue, not individual test bugs."
  },
  "metadata": {
    "workflow_name": "release_build",
    "repo": "fabric2/core-service",
    "branch": "main",
    "sha": "a3f7d2c1",
    "run_id": "12345678"
  },
  "version": "1.0"
}
```

**Failure categories:**

| Category | Meaning |
|----------|---------|
| `test_failure` | Unit/integration test assertion failures |
| `compilation_error` | Build-time compilation errors |
| `dependency_issue` | Maven/Gradle/npm resolution failures |
| `timeout` | Workflow or step exceeded time limit |
| `resource_exhaustion` | OOM kill, disk full, CPU throttle |
| `infra_flake` | Transient infra/network issue |
| `config_error` | Misconfigured environment, secrets, or YAML |
| `unknown` | Could not determine root cause with confidence |

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `GITHUB_TOKEN` | — | **Required.** PAT with `actions:read` + `pull-requests:write` |
| `GITHUB_API_URL` | `https://api.github.com` | API base URL — auto-set by Actions via `${{ github.api_url }}` |
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama service URL |
| `OLLAMA_MODEL` | `gemma3:27b-it-qat` | Model name |
| `MAX_LOG_TOKENS` | `80000` | Max tokens sent to Gemma (80K → ~320K chars) |
| `BUILD_TIME_THRESHOLD_MINUTES` | `20` | Flag builds that exceed this duration |
| `SLACK_WEBHOOK_URL` | — | Optional Slack webhook for notifications |
| `REQUESTS_CA_BUNDLE` | — | Enterprise CA cert path (GitHub Enterprise Server) |

## Enterprise Configuration

This system is built for **GitHub Enterprise Server** and **air-gapped GDC clusters**:

- **`GITHUB_API_URL`** is used everywhere — never hardcoded `api.github.com`
- **`${{ github.api_url }}`** is passed automatically to the workflow, resolving correctly for both `github.com` and GHE Server (`https://ghe.corp.com/api/v3`)
- **Runner networking**: three supported scenarios — runner inside GDC (ClusterIP), peered VM (NodePort), or dedicated GDC runner
- **Corporate proxy**: set `NO_PROXY` to include Ollama hostname so inference traffic bypasses the proxy
- **Enterprise CA cert**: set `REQUESTS_CA_BUNDLE` env var if runners get SSL errors

See [ENTERPRISE_ACCESS_AND_5DAY_PLAN.md](ENTERPRISE_ACCESS_AND_5DAY_PLAN.md) for the full access guide, PAT approval process, firewall rules, and 5-day POC execution plan.

## Design Principles

| Principle | Implementation |
|-----------|----------------|
| **Never crash the build** | Always writes a fallback report and exits 0 — the build outcome is never blocked by RCA |
| **Data sovereignty** | Ollama runs on-cluster (ClusterIP); build logs never leave the org |
| **Resilient inference** | 3 retries with exponential backoff; health check before sending 80K tokens |
| **Enterprise-first** | `GITHUB_API_URL` env var everywhere; supports GHE Server + github.com without code changes |
| **Focused analysis** | Error pattern extraction + head/tail trimming sends only the most relevant log sections |
| **Structured output** | Always returns JSON — machine-readable for dashboards, ticketing, and ML pipelines |
